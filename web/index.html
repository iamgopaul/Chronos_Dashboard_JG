2. Column mapping

WRDS / Compustat-style files: Preview columns auto-fills datadate, gvkey, ROA (or ROE), and QE when present. costat is a status code, not a numeric target.

Research protocol

Matches the project sequence: WRDS fundq-style extracts → long panel with item_id=gvkey, timestamp=datadate, target=ROA or ROE (ratio scale, e.g. 0.05 not 5). Chronos-Bolt Base is zero-shot (no fine-tuning). Outputs include probabilistic bands (P10–P90) and the same error metrics cited in the proposal (MASE, sMAPE, WQL).

  • Quarterly alignment: set frequency to QE; censor M&A targets in WRDS before export when following the censored-series rule (dlrsn).
  • Chronos context must include years before the evaluation window—do not narrow Configuration dates to 2023–2024 only (that drops pre-2023 history). Leave dates empty or use the full range from Preview; rolling one-step with N=8 evaluates the last eight quarters in your filtered series (often 2023–2024 if the panel ends in 2024).
  • NLI, Bai–Perron/ruptures, and Auto-ARIMA benchmarks run outside this app (R/Python/SPSS per your Data Preparation & ARIMA docs); export the cleaned CSV and use this UI for the Chronos leg of the tournament.
